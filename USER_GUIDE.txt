1. SYSTEM OVERVIEW
================================================================================

NRP CRM is a comprehensive Client Relationship Management platform designed
specifically for wealth management firms. The system streamlines:

- Client Onboarding: Automated document collection and verification
- Portfolio Management: Real-time holdings tracking and performance analytics
- Communication: Family-based messaging with privacy controls
- Task Management: 7-state Kanban workflow for operations
- Compliance: Meeting notes, reminders, and audit trails

KEY FEATURES:
- Role-based access control (Admin, RM, Client)
- Privacy filtering (Internal vs. External content)
- Automated reminders and notifications
- Document verification workflow
- Real-time portfolio tracking
- Multi-RM family support

2. USER ROLES & ACCESS LEVELS
================================================================================

ADMIN (Financial Advisor)
-------------------------
Access: Full system access to all families and features
Primary Functions:
- View system-wide wealth metrics and AUM
- Manage all client onboarding checklists
- Verify documents for all families
- Access all messages (internal and external)
- Create and manage email templates
- Oversee all meeting notes and reminders
- View all tasks across the organization

Key Pages:
- Admin Dashboard: System-wide overview
- Onboarding Master: All checklists
- Email Templates: Communication management
- Meeting Notes: All client meetings
- Admin Reminders: System-wide reminders

RELATIONSHIP MANAGER (RM)
--------------------------
Access: Assigned families only
Primary Functions:
- Manage assigned client portfolios
- Track onboarding for assigned families
- Communicate with assigned clients
- Create and complete tasks for assigned families
- Schedule meetings and manage calendar
- Set personal reminders

Key Pages:
- RM Dashboard: Assigned families overview
- Calendar: Schedule meetings and events
- Communications: Messages for assigned families
- Tasks: Family-specific task management
- RM Reminders: Personal reminders

FAMILY/CLIENT
-------------
Access: Own family data only, external content only
Primary Functions:
- View own portfolio and holdings
- Track onboarding progress
- Upload required documents
- View external messages (internal notes hidden)
- Track meetings (client-visible only)
- View transactions and performance

Key Pages:
- Client Dashboard: Portfolio overview
- Documents: Upload and track documents
- Communications: External messages only
- Meetings: Client-visible meeting notes

4.5 MEETING NOTES
-----------------
URL: /admin/meeting-notes

Purpose: Document client meetings with privacy controls and action items

User Flow:
1. VIEW MEETING METRICS
   - StatCards: Total Meetings, This Week, Action Items, Overdue Actions
   - Track meeting activity

2. CREATE NEW MEETING NOTE
   a) Click "New Meeting Note"
   b) Fill Meeting Details:
      - Select family
      - Choose meeting type (Onboarding, Review, Planning, etc.)
      - Set meeting date and time
      - Select status (Scheduled, Completed, Cancelled)

   c) Set Privacy Level (CRITICAL):
      - Internal Only: Hidden from clients completely
      - Summary Only: Client sees summary field only
      - Client Visible: Client sees full notes

   d) Add Participants:
      - Select attendees
      - Assign roles (Primary Client, Family Member, RM, Admin)
      - Mark attendance status

   e) Document Meeting:
      - Enter discussion points (bulleted list)
      - Document decisions made
      - Add internal notes (always hidden from clients)
      - Write client summary (if privacy = Summary Only)

   f) Create Action Items:
      - Add action item description
      - Assign to team member
      - Set due date
      - System auto-creates reminder

   g) Add Next Steps:
      - List follow-up actions
      - Schedule next meeting if needed

3. REVIEW EXISTING NOTES
   - Filter by family, date, type, privacy level
   - Search by content
   - Sort by date (newest first)

4. EDIT MEETING NOTES
   - Click "Edit" on meeting card
   - Modify any fields
   - Update action item status
   - Add additional notes

5. TRACK ACTION ITEMS
   - View all action items from meetings
   - Filter by status (Pending, In Progress, Completed)
   - Monitor overdue items
   - Mark as completed when done

Best Practices:
- Set correct privacy level based on content sensitivity
- Use "Internal Only" for sensitive discussions
- Use "Summary Only" for standard meetings with internal analysis
- Use "Client Visible" for transparent meetings
- Create action items during meeting
- Follow up on overdue action items
- Document decisions clearly

PRIVACY CONTROLS (CRITICAL):
- Internal notes are ALWAYS hidden from clients
- Action items shown only if assigned to client
- Summary Only shows summary field only to clients
- Client Visible shows full notes to clients
- Double-check privacy level before saving

4.6 ADMIN REMINDERS
-------------------
URL: /admin/reminders

Purpose: System-wide reminder management with recurring patterns

User Flow:
1. VIEW REMINDER METRICS
   - StatCards: Overdue, Due Today, Due This Week, Completed
   - Prioritize overdue items

2. REVIEW REMINDER LIST
   - Grouped by status (Overdue, Today, This Week, Later, Completed, Snoozed)
   - Priority badges (Low, Medium, High, Urgent)
   - Family name shown
   - Due date displayed

3. CREATE NEW REMINDER
   a) Click "New Reminder"
   b) Basic Details:
      - Enter reminder title
      - Add description (optional)
      - Select family (optional)
      - Choose priority (Low, Medium, High, Urgent)

   c) Due Date:
      - Set due date and time

   d) Recurrence (optional):
      - Enable recurring
      - Select pattern (Daily, Weekly, Monthly, Yearly)
      - Set end condition:
        * End by date
        * End after N occurrences
        * Never end

   e) Assignment:
      - Assign to user or self
      - System tracks assigned reminders

4. MANAGE REMINDERS
   a) Complete Reminder:
      - Click "Mark Complete"
      - Add completion notes (optional)
      - System archives completed reminder

   b) Snooze Reminder:
      - Click "Snooze"
      - Select preset: 1 hour, 4 hours, Tomorrow, Next week
      - Or choose custom date/time
      - Reminder hidden until snooze expires

   c) Edit Reminder:
      - Click "Edit"
      - Modify any fields
      - Save changes

   d) Delete Reminder:
      - Click "Delete"
      - Confirm deletion

5. FILTER AND SEARCH
   - Filter by priority
   - Filter by status
   - Filter by family
   - Search by title/description

Best Practices:
- Check overdue reminders daily
- Set appropriate priority levels
- Use recurring reminders for regular tasks
- Snooze instead of ignoring
- Add notes when completing
- Review "Due Today" first thing each morning

AUTOMATED REMINDERS:
System automatically creates reminders for:
- Document uploaded (verify within 2 days)
- Document rejected (follow up with client)
- Checklist completed (schedule onboarding meeting)
- Meeting scheduled (1 day before, 1 hour before)
- Action items (on due date)

4.14 TASKS (Admin & RM)
-----------------------
URL: /tasks

Purpose: 7-state Kanban board for operational task management

User Flow:
1. VIEW KANBAN BOARD
   - 7 columns shown:
     1. To Do
     2. In Progress
     3. In Review
     4. Pending Document from Client
     5. Waiting on Client
     6. Blocked
     7. Done
   - Tasks shown as cards in columns
   - Count displayed in column header

2. VIEW TASK DETAILS
   Each task card shows:
   - Task title
   - Family name
   - Priority badge (Low, Medium, High, Urgent)

   - Assigned user name

3. CREATE NEW TASK
   a) Click "New Task"
   b) Task Details:

      - Select family
      - Choose context type (Onboarding, Compliance, Document, Meeting, General)

   c) Assignment and Priority:
      - Assign to user or team

   d) Initial Status:
      - New tasks start in "To Do" column

   e) Save Task:
      - Click "Create Task"
      - Task appears in To Do column

   - Click and hold task card
   - Drag to target column
   - Release to drop
   - Status updates automatically
   - For "Blocked" status: System prompts for blocking reason
   - For "Waiting on Client": System prompts for wait reason
   - For "Pending Document": System prompts for document type

5. EDIT TASK
   - Click task card to open detail view
   - Click "Edit"
   - Modify any fields
   - Save changes

6. COMPLETE TASK
   - Drag to "Done" column
   - System records completion time
   - Task archived after 30 days

7. FILTER TASKS
   - Filter by priority
   - Filter by family
   - Filter by assigned user
   - Filter by due date

Best Practices:
- Update task status regularly
- Use appropriate context types
- Set realistic due dates
- Provide clear blocking reasons
- Move to "In Progress" when starting work
- Use "In Review" for tasks awaiting approval
- Use "Pending Document" when waiting for client uploads
- Use "Waiting on Client" for client responses
- Use "Blocked" for external dependencies
- Keep "Done" column clean (auto-archives after 30 days)

ROLE-BASED ACCESS:
- Admin: All tasks
- RM: Tasks for assigned families only
- Family: No access (internal tool)

TASK STATES EXPLAINED:
- To Do: New tasks, not yet started
- In Progress: Currently being worked on
- In Review: Awaiting review/approval by supervisor
- Pending Document from Client: Waiting for client to upload document
- Waiting on Client: Waiting for client response or decision
- Blocked: Cannot proceed due to external dependency
- Done: Task completed

5. FEATURE-SPECIFIC WORKFLOWS
================================================================================

5.1 CLIENT ONBOARDING WORKFLOW (End-to-End)
--------------------------------------------

Step 1: Create Onboarding Checklist (Admin)
- Navigate to Admin Dashboard > Onboarding
- Click "New Checklist"
- Select family
- Choose service type (NRP Light or NRP 360)
- Indicate if KYC already done
- System generates conditional checklist automatically
- Required documents adapt based on selections

Step 2: Invite Client for Document Upload (Admin/RM)
- Navigate to checklist detail page
- Click "Generate Temp Access" (future feature)
- System generates secure token with expiry
- Send access link to client via email
- Client can upload documents without login

Step 3: Client Uploads Documents
- Client receives email with secure link
- Clicks link to access upload portal
- Views required documents list
- Uploads each document:
  * Click "Upload" on required item
  * Select file (PDF, JPG, PNG)
  * Confirm upload
  * Status changes to "Pending"
- Repeats for all required documents

Step 4: Admin/RM Verifies Documents

  * Or click "Reject" with clear reason

- Verified: Green badge, counts toward progress
- Rejected: Red badge, shows reason, awaits re-upload

- Client sees rejection reason
- Uploads corrected document
- Status changes to "Pending" again
- Awaits re-verification

Step 6: Complete Checklist
- When all documents verified (100% completion)
- System auto-advances to next stage:
  * KYC & Documents → Data Input
  * Data Input → Execution
  * Execution → Completed
- System creates automated reminder to schedule onboarding meeting

Step 7: Onboarding Meeting
- Schedule meeting via Calendar
- Document meeting in Meeting Notes
- Set privacy level appropriately
- Create action items for next steps

Step 8: Client Activation
- Checklist marked as "Completed"
- Client gains full access to dashboard
- Portfolio setup begins
- Regular client communications commence

TIMELINE: Typical onboarding takes 7-14 days depending on client responsiveness

5.6 DOCUMENT VERIFICATION WORKFLOW
-----------------------------------

For Admin/RM:
1. Receive notification of document upload (automated reminder)
2. Navigate to Onboarding > Checklist Detail
3. Locate uploaded document in Verification Panel
4. Click "View" to preview document
5. Check document for:
   - Clarity and readability
   - Completeness (all pages present)
   - Correct document type
   - Valid signatures (if required)
   - Current information (not expired)
6. Decision:
   a) If acceptable:
      - Click "Verify"
      - Status changes to "Verified" (green badge)
      - Counts toward completion progress
      - Reminder marked as complete

   b) If not acceptable:
      - Click "Reject"
      - Provide clear, specific rejection reason
        Examples:
        - "Document is not clear, please upload higher quality scan"
        - "Missing signature on page 3"
        - "Document expired, please provide current version"
        - "Wrong document type uploaded"
      - Status changes to "Rejected" (red badge)
      - Reason shown to client
      - Client receives notification (future feature)
7. Repeat for all uploaded documents

For Client (Re-upload):
1. View rejected document
2. Read rejection reason carefully
3. Prepare corrected document
4. Upload new version
5. Status changes to "Pending"
6. Await re-verification

Best Practices:
- Verify within 2 business days
- Provide clear, actionable rejection reasons
- Check all pages of multi-page documents
- Verify signatures and dates
- Ensure document matches required type
- Be professional and helpful in rejection notes

7.1 COMMON ISSUES
-----------------

Issue: Cannot login
Solutions:
- Check username and password (case-sensitive)
- Try demo accounts:
  * Admin: admin / admin123
  * RM: rm / rm123
  * Client: sharma / demo123
- Clear browser cache and try again
- Try different browser
- Contact system administrator

Issue: Don't see expected families in list
Solutions:
- Check your role: RMs see only assigned families
- Admin should see all families
- Clients see only their own family
- Check filters applied to list
- Refresh page

Issue: Internal messages showing to client
Solutions:
- Check message "is_internal" flag
- System auto-tags based on participants
- No client participants = Internal
- With client participants = External
- Review message privacy settings

Issue: Document upload fails
Solutions:
- Check file size (< 10 MB recommended)
- Check file type (PDF, JPG, PNG supported)
- Try different browser
- Clear browser cache
- Check internet connection
- Contact support if persists

Issue: Reminder not showing
Solutions:
- Check status filters (not filtered out?)
- Check if snoozed (check "Snoozed" section)
- Check if completed (check "Completed" section)
- Check date filters
- Refresh page

Issue: Task not moving on Kanban board
Solutions:
- Check if drag-and-drop is enabled
- Try clicking task and using status dropdown
- Refresh page
- Check browser compatibility (Chrome, Firefox, Safari recommended)
- Try desktop if on mobile

Issue: Meeting notes privacy not working
Solutions:
- Verify privacy level set correctly
- Internal Only = Completely hidden from clients
- Summary Only = Only summary shown to clients
- Client Visible = Full notes shown
- Check if viewing as correct role
- Logout and login again to refresh

Issue: Portfolio values not updating
Solutions:
- Check if transactions recorded
- System calculates automatically
- Refresh page
- Check if holding exists
- Contact RM/Admin for corrections

7.2 FREQUENTLY ASKED QUESTIONS
-------------------------------

Q: How do I add a new family?
A: (Admin only) Navigate to Admin Clients > New Family (future feature)
   Current MVP uses sample data only

Q: How do I assign an RM to a family?
A: (Admin only) Navigate to Family details > Assign RM (future feature)
   Current MVP has fixed RM assignments

Q: Can clients see internal messages?
A: No. Internal messages are automatically hidden from clients. Only messages
   with client participants are visible to families.

Q: How long does onboarding take?
A: Typical onboarding takes 7-14 days depending on:
   - Document availability from client
   - Verification speed by Admin/RM
   - Completeness of initial uploads
   - Service type (NRP Light vs NRP 360)

Q: What's the difference between NRP Light and NRP 360?
A: NRP Light: Basic wealth management services with simplified documentation
   NRP 360: Comprehensive wealth management with full documentation requirements

Q: Can I delete a message?
A: Not currently. Messages are permanent for audit trail purposes.
   Future feature may allow archiving.

Q: How do I generate reports?
A: Reporting feature coming in future update. Currently export data manually
   from each page.

Q: Can I customize email templates?
A: Yes (Admin only). Navigate to Admin Communications to create, edit, and
   delete email templates.

Q: How do I change my password?
A: Navigate to user profile (future feature). Current MVP has fixed passwords
   for demo accounts.

Q: Can I access from mobile?
A: Yes. The platform is mobile-responsive. Use any modern mobile browser.

Q: What file types are supported for documents?
A: Currently: PDF, JPG, PNG
   Future: DOCX, XLSX may be added

Q: How do I know if a client uploaded a document?
A: Automated reminder created on document upload. Check Reminders page or
   Onboarding checklist for pending verifications.

Q: Can I schedule recurring meetings?
A: Not currently. Create individual meeting events. Recurring events feature
   coming in future update.

Q: How do I export portfolio data?
A: Export feature coming in future update. Currently take screenshots or
   manually copy data.

Q: What happens to completed tasks?
A: Tasks in "Done" column are archived after 30 days. They remain accessible
   in completed tasks view.

Q: Can I assign multiple RMs to one family?
A: Yes. Multi-RM support is built-in. All assigned RMs see all messages for
   that family.

Q: How do I mark a review as overdue?
A: System automatically calculates based on "Next Review Date" field. Update
   the field to change due date.

Q: Can clients create tasks?
A: No. Tasks are internal tools for Admin/RM only. Clients cannot access the
   Tasks page.

Q: How do I filter messages by date?
A: Use search functionality and date filters on Communications page.

Q: What is XIRR?
A: Extended Internal Rate of Return - a measure of portfolio performance that
   accounts for the timing and amount of cashflows. Shows annualized return.

Q: How are asset allocation percentages calculated?
A: System calculates based on current value of each asset class divided by
   total portfolio value. Updates automatically as holdings change.

Q: Can I undo a document verification?
A: Not currently. If verified incorrectly, reject the document with explanation
   and client will need to re-upload.

Q: How do I add a new document type to checklist?
A: (Admin only) Modify checklist template in system configuration (future
   feature). Current MVP has fixed document types.

Solution: Convert to PDF, JPG, or PNG and try again

Error: "Failed to save"
Meaning: Data couldn't be saved to localStorage
Solution: Check browser storage settings, clear cache, try again

Error: "Session expired"

Solution: Login again
